This is the technical documentation of Atarilegend. You can find the database structure and more info to give you a better insight of the project.


Contents Menu section


Menu section




This screenshot is a visual representation of the current table structure used for the menu section. This ER diagram shows exactly how all tables are connected to each other. This section is ofcourse linked to the other sections of the database. Check the other ER diagrams to get a better idea. Below you will find a complete description of each table used here.

Layout of tables :

menu_type : This table connects all menu sets and even menu disks to a certain type of menu
menu_types_main : This table contains the actual menu types eg. demo menu, game menu, tool menuâ€¦
menu_set : Many crews made different "sets" of menu disks. One example would be the Pompey Pirates, they had 2 sets, the actual 'Pompey Pirates' set and another one named "Krappy Compacts".
menu_sets_order : This table is needed specifically for putting the discs in the correct way on screen. We need to know how many discs are released, which number they start and end, gaps, how they are ordered ...
menu_disk : This is the main table for the menu disks. Every menu disk belongs to a set, hence the set id.
menu_disk_title : Menu disks can contain all kinds of software titles. These titles are stored in the menu_disk_title table and are linked to the actual menu disk using the menu_disk_id field. This title table also contains menu_disk_title_type field. This means that, even if a menu disk is linked to a 'game' type, each title individually has its own type. So, a game menu disk, can also contain a doc for example.
menu_disk_title_game : All the game titles are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_tools : All the tool titles are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_demo : All the demo titles are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_music : All the music titles are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_doc_games : All the (doc discs with) game docs are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_doc_games : All the (doc discs with) tool docs are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_various : All titles not belonging to any type of the above are contained in this table. We have seperate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_set : This table contains the sets of titles. For example, let's say a menu disk contains the game title 'Gods'. However, this title is normally spread over 2 disks. Using this 'set' table, we can link a title over 2 discs.
screenshot_menu : This is the screenshot cross table linking the menu disks to the screenshot main table (for more detail, see the screenshots section).
menu_user_comments : This is the comments cross table linking the menu disks to the user comments main table (for more detail, see the comments section).
menu_disk_download : This table contains the actual files for download of each menu disk.
menu_disk_submenu : It sometimes happens that a menudisk is some kind of submenu of a totally different other menu disk. Using the cross table we can link menu disks to other menu disks.
menu_disk_state : The main states table containing info like 'corrupt', 'complete' ...
menu_disk_credits : This table contains the individual id's of the people who were responsible for the intro of the menudisk. This is a cross table between the menu disk and the individuals table.
