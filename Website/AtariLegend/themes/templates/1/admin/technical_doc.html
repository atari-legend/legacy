Menu section




The above Entity Relation (ER) diagram shows exactly how all of the tables are connected to each other. This data is linked to the other sections of the database. Look at the other ER diagrams to get a better idea. There is a complete description of each table used for the menu disks below.

Layout of tables :

menu_type : This table connects all menu sets and even menu disks to a certain type of menu
menu_types_main : This table contains the actual menu types e.g. demo menu, game menu, tool menu…
menu_set : Many crews made different "sets" of menu disks. One example would be the Pompey Pirates as they had 2 sets; the actual 'Pompey Pirates' set and another one named "Krappy Compacts".
menu_sets_order : This table is needed specifically for ordering the disks in the correct way. We need to know how many disks were released, which number they start and end at, any known gaps and how they are ordered.
menu_disk : This is the main table for the menu disks. Every menu disk belongs to a set, hence the set identifier.
menu_disk_title : Menu disks can contain all kinds of software titles. These titles are stored in the menu_disk_title table and are linked to the actual menu disk using the menu_disk_id field. This title table also contains menu_disk_title_type field. This means that, even if a menu disk is linked to a 'game' type, each title individually has its own type. So, a game menu disk, can also contain a doc for example.
menu_disk_title_game : All the game titles are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_tools : All the tool titles are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_demo : All the demo titles are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_music : All the music titles are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_doc_games : All the (doc disks with) game docs are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_doc_tools : All the (doc disks with) tool docs are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_various : All titles not belonging to any type of the above are contained in this table. We have separate table for each type of title so we can enhance search features on the site and for data consistency.
menu_disk_title_set : This table contains the sets of titles. For example, let's say a menu disk contains the game title 'Gods'. However, this title is normally spread over 2 disks. Using this 'set' table, we can link a title over 2 discs.
screenshot_menu : This is the screenshot cross table linking the menu disks to the screenshot main table (for more detail, see the screenshots section).
menu_user_comments : This is the comments cross table linking the menu disks to the user comments main table (for more detail, see the comments section).
menu_disk_download : This table contains the actual files for download of each menu disk.
menu_disk_submenu : It sometimes happens that a menu disk is some kind of submenu of a totally different other menu disk. Using the cross table we can link menu disks to other menu disks.
menu_disk_state : The main states table containing info like 'corrupt', 'complete' ...
menu_disk_credits : This table contains the individual id's of the people who were responsible for the intro of the menu disk. This is a cross table between the menu disk and the individuals table.
menu_disk_year : This table contains the release year of the menu disk.
menu_disk_title_author : This table is used to store the credits for each menu disk. Such as the crackers, hackers, artists etc.


Doc disk section




Overhere we see a visual overview of how the doc tables are linked to the menu section by use of doc disks. As you also can see, not only doc disks are linked to the doc table. Other sections will be linked as wel. Some examples :

- Let's say we are dealing with a SEWER DOC DISK containing a manual of "Xenon 2" and a cheat code for "Supercars". Both docs have got an entry in the menu_disk_title_doc_games, in the doc_disk_game and in the doc table. There can be more docs of the same game (even with the same doc type!). These docs wil have the category "doc disk game".

- The game "Xenon 2" by itself can also be linked to the doc table. What if we are dealing with the official manual of the game. This has nothing to do with a doc disk. So a cross table between the game table and the doc table is used, in this case the doc_game table.

Layout of tables :

doc_disk_game : This is the cross table linking the doc disk title (in this case a game doc) with the actual doc table.
doc_disk_tool : This is the cross table linking the doc disk title (in this case a tool doc) with the actual doc table.
doc_game : This is the cross table linking the game table with the doc table.
doc : Main documents table containing all docs used on the site
doc_type : The type of document we are dealing with eg. a cheatcode, a walkthrough, a manual ...
doc_categort : The category this document belongs to eg. doc disk game, doc disk tool, doc game, doc tool ...


Crew section




Another big, complex part of the website. These table contain all data on the crews and individuals. Their complete relationship with eachother and their work.

Layout of tables :

crew : The main crew table containing all the crews.
sub_crew : Crews sometimes consist of subcrews.
crew_individual : The cross table linking the individuals(hackers, coders, musicians…) to their crew.
individuals : The main individual table.
crew_menu_prod, crew_demo_prod, ind_demo_prod, ind_menu_prod : The cross tables linking the crews and individuals to their work. Every crew or individual has got their own productions, these are always demos or menus. Games are dealt with in the game tables structure.


Individual section




The following tables will stock the individuals. Sceners, game/demo creators, musicians … These tables are used in various structures all over the DB.

Layout of tables :

individuals : The main individuals table will hold the individual names AND nicknames.
individual_nicks : A cross table linking the individual with its nick name(s).
individual_text : This table contains extra details of the individual. eg history, email ...


Download section




The following tables will contain all download info.

Layout of tables :

download_main : this is the main table which holds all downloads from the site. These can be menu downloads, game downloads, demo downloads...
game_download : This is the cross table linking the download to the game, thus we are dealing with a game download
game : The main game table
game_download_details : This table contains some extra details of the download
game_download_options : This is the cross table to the download options table.
download_options : This is the 'label' table. This table can hold 'labels' like : PAL, NTSC, HD, Crack, Menu, Original, Patch...
game_download_tos : The cross table to the TOS version table
tos_version : Table containing al TOS versions
game_download_lingo : cross table to the main language table (lingo) which contains the actual language of the game download
lingo : main language table
game_download_set : cross table used for games spanned over more than 1 disk
game_download_crew : cross table to the crew table - this one is filled if the download is a crew release
game_download_individual : cross table to the individual table - this one is filled if the download is an individual release. Also contains the author_type_id to link to the author_type table explaining the job of the individual
game_download_trainer : a cross table to the trainer table containing all the trainer options of the download
trainer_options : table containing all types of trainers
game_download_intro : cross table to the demo table in case the download contains a cracktro
demo : main demo table
game_download_menu : cross table to the menu section, should this download be part of a menu disk
download_format : cross table to the format table
format : main format table (Kyro, SCP, MSA, Past, Files ...)
game_download_info : this table contains info on who downloaded what. This is an updated version of the old table in which I have added the game_download_id, making it possible to see which game was downloaded.
